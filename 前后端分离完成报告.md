### 2. 数据库设计

#### 数据表（5张）

**users 表**
```sql
- id (主键)
- username (用户名，唯一)
- password (密码，加密)
- name (姓名)
- email (邮箱，唯一)
- role (角色：admin/student)
- avatar (头像URL)
- studentId (学号)
- major (专业)
- grade (年级)
- phone (手机号)
- createdAt, updatedAt
```

**activities 表**
```sql
- id (主键)
- title (标题)
- description (描述)
- category (分类)
- cover (封面)
- location (地点)
- startTime (开始时间)
- endTime (结束时间)
- maxParticipants (最大人数)
- participants (当前人数)
- organizer (主办方)
- tags (标签，JSON)
- status (状态)
- createdBy (创建者ID)
- createdAt, updatedAt
```

**registrations 表**
```sql
- id (主键)
- userId (用户ID)
- activityId (活动ID)
- status (状态：registered/cancelled/attended)
- createdAt, updatedAt
- 唯一索引：(userId, activityId)
```

**favorites 表**
```sql
- id (主键)
- userId (用户ID)
- activityId (活动ID)
- createdAt, updatedAt
- 唯一索引：(userId, activityId)
```

**comments 表**
```sql
- id (主键)
- activityId (活动ID)
- userId (用户ID)
- content (内容)
- createdAt, updatedAt
```

#### 数据关联
- User ↔ Activity (多对多，通过 Registration)
- User ↔ Activity (多对多，通过 Favorite)
- User → Comment (一对多)
- Activity → Comment (一对多)
- User → Activity (一对多，创建者)

### 3. API接口（30+个）

#### 用户相关（9个）
- `POST /api/users/register` - 用户注册
- `POST /api/users/login` - 用户登录
- `GET /api/users/me` - 获取当前用户
- `GET /api/users/students` - 获取学生列表（管理员）
- `GET /api/users/:id` - 获取用户详情
- `POST /api/users` - 创建用户（管理员）
- `PUT /api/users/:id` - 更新用户
- `DELETE /api/users/:id` - 删除用户（管理员）
- `POST /api/users/change-password` - 修改密码

#### 活动相关（6个）
- `GET /api/activities` - 获取活动列表（支持分页、搜索、筛选）
- `GET /api/activities/:id` - 获取活动详情
- `POST /api/activities` - 创建活动（管理员）
- `PUT /api/activities/:id` - 更新活动（管理员）
- `DELETE /api/activities/:id` - 删除活动（管理员）
- `GET /api/activities/statistics` - 获取统计数据

#### 报名相关（5个）
- `POST /api/registrations` - 报名活动
- `DELETE /api/registrations/:activityId` - 取消报名
- `GET /api/registrations/user/:userId` - 获取用户报名
- `GET /api/registrations/activity/:activityId` - 获取活动报名
- `GET /api/registrations/stats/all` - 获取报名统计（管理员）

#### 收藏相关（4个）
- `POST /api/favorites` - 收藏活动
- `DELETE /api/favorites/:activityId` - 取消收藏
- `GET /api/favorites/user/:userId` - 获取用户收藏
- `GET /api/favorites/check/:activityId` - 检查是否收藏

#### 评论相关（3个）
- `POST /api/comments` - 发表评论
- `GET /api/comments/activity/:activityId` - 获取活动评论
- `DELETE /api/comments/:id` - 删除评论

## 🎯 核心优势

### 1. 数据持久化
- ❌ 之前：数据存储在浏览器localStorage，清除缓存即丢失
- ✅ 现在：数据存储在MySQL数据库，永久保存

### 2. 多用户协作
- ❌ 之前：每个用户看到的数据独立，无法共享
- ✅ 现在：所有用户共享同一数据库，实时同步

### 3. 数据安全
- ❌ 之前：前端数据可被随意修改
- ✅ 现在：后端验证+权限控制，数据安全可靠

### 4. 业务逻辑
- ❌ 之前：复杂逻辑在前端实现，难以维护
- ✅ 现在：业务逻辑在后端统一处理，易于扩展

### 5. 性能优化
- ❌ 之前：所有数据加载到前端，占用内存
- ✅ 现在：按需加载，支持分页，性能更好

## 🚀 使用方式

### 快速启动（3步）

1. **初始化数据库**
```bash
cd backend
npm install
npm run init-db
```

2. **启动后端**
```bash
npm run dev
```

3. **启动前端**（新终端）
```bash
cd ..
npm run dev
```

### 访问系统

- 前端：http://localhost:5173
- 后端：http://localhost:3000
- 测试账号：
  - 管理员：admin / 123456
  - 学生：student1 / 123456

## 📝 注意事项

### 1. 环境要求
- Node.js >= 16.0.0
- MySQL >= 5.7
- npm 或 yarn

### 2. 配置文件
- `backend/.env` - 后端环境变量（必须配置数据库密码）
- `.env` - 前端环境变量（可选，有默认值）

### 3. 数据库
- 需要先创建数据库：`campus_activities`
- 运行 `npm run init-db` 初始化表和数据

### 4. 端口占用
- 后端默认：3000
- 前端默认：5173
- 如有冲突，修改配置文件
