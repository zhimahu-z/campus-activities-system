# 用户管理系统与真实数据统计说明

## 🎯 功能概述

本次更新实现了完整的用户管理系统，管理后台的所有数据统计现在基于真实的用户报名行为，而不是模拟数据。

## 📋 核心功能

### 1. 用户管理系统

#### 默认账号
系统初始化时只有一个管理员账号：
- **用户名**: `admin`
- **密码**: `123456`
- **角色**: 管理员（admin）

#### 创建用户功能
管理员可以创建学生账号，需要填写以下信息：
- ✅ 用户名（用于登录，3-20个字符）
- ✅ 密码（至少6个字符）
- ✅ 姓名（真实姓名）
- ✅ 学号
- ✅ 专业
- ✅ 年级（2024级/2023级/2022级/2021级）
- ✅ 邮箱（必填，格式验证）
- ⭕ 手机号（可选）

**创建的账号特点**：
- 默认角色为**学生**（student）
- 只有学生权限，无法访问管理后台
- 自动生成头像（基于用户名）
- 记录创建时间

### 2. 真实数据统计

#### 报名数据统计规则
- ✅ 只有通过管理员创建的学生账号报名才计入统计
- ✅ 每个活动的参与人数 = 报名该活动的学生账号数量
- ✅ 总参与人数 = 所有活动的报名总数
- ✅ 平均参与率 = 基于真实报名数据计算

#### 初始状态
- 所有活动的初始参与人数为 **0**
- 管理后台显示的数据都是 **0**
- 只有创建学生账号并报名后，数据才会增加

### 3. 数据持久化

所有数据存储在浏览器的 `localStorage` 中：

```javascript
// 用户数据
localStorage.setItem('allUsers', JSON.stringify(users))

// 每个用户的报名记录
localStorage.setItem(`registrations_${userId}`, JSON.stringify(registrations))

// 活动数据
localStorage.setItem('activities', JSON.stringify(activities))
```

## 🔧 技术实现

### 1. 用户Store (user.js)

#### 新增功能
```javascript
// 初始化用户列表
initUsers()

// 创建用户
createUser(userData)

// 获取所有学生
getAllStudents()

// 删除用户
deleteUser(userId)

// 更新用户信息
updateUser(userId, userData)

// 获取用户的报名记录
getUserRegistrations(userId)

// 获取所有用户的报名统计
getAllRegistrationsStats()
```

#### 报名统计逻辑
```javascript
const getAllRegistrationsStats = () => {
  const stats = {}
  allUsers.value.forEach(user => {
    if (user.role === 'student') {
      const userRegs = getUserRegistrations(user.id)
      userRegs.forEach(activityId => {
        stats[activityId] = (stats[activityId] || 0) + 1
      })
    }
  })
  return stats
}
```

### 2. 活动Store (activity.js)

#### 修改内容
```javascript
// 初始化时所有活动参与人数为0
participants: 0

// 更新活动的参与人数
updateParticipantsCount(registrationStats)

// 获取活动的真实参与人数
getActivityParticipants(activityId, registrationStats)
```

### 3. 管理后台 (Dashboard.vue)

#### 核心计算属性
```javascript
// 获取真实的报名统计数据
const registrationStats = computed(() => userStore.getAllRegistrationsStats())

// 总参与人数（基于真实报名）
const totalParticipants = computed(() => {
  return Object.values(registrationStats.value).reduce((sum, count) => sum + count, 0)
})

// 热门活动（基于真实报名人数排序）
const hotActivities = computed(() => {
  return activityStore.activities
    .map(a => ({
      ...a,
      participants: registrationStats.value[a.id] || 0
    }))
    .sort((a, b) => b.participants - a.participants)
    .slice(0, 5)
})
```

## 📝 注意事项

### 1. 数据清理
- 删除用户会清除其所有数据
- 删除活动不会影响用户数据
- 清空localStorage会丢失所有数据

### 2. 密码安全
- 当前密码明文存储（仅用于演示）
- 生产环境需要加密存储
- 建议使用强密码

### 3. 浏览器兼容性
- 需要浏览器支持localStorage
- 数据存储在本地，不同浏览器不共享
- 清除浏览器数据会丢失所有记录

### 4. 性能考虑
- 用户数量较多时，统计计算可能较慢
- 建议定期清理无用数据
- 可以考虑添加缓存机制
