## 📝 详细步骤

### 第一步：准备环境

#### 1. 安装MySQL

**Windows**:
- 下载：https://dev.mysql.com/downloads/mysql/
- 安装后记住root密码

**Mac**:
```bash
brew install mysql
brew services start mysql
```

**Linux**:
```bash
sudo apt-get install mysql-server
sudo systemctl start mysql
```

#### 2. 创建数据库

打开MySQL命令行：
```bash
mysql -u root -p
```

创建数据库：
```sql
CREATE DATABASE campus_activities CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

### 第二步：配置后端

#### 1. 进入后端目录
```bash
cd backend
```

#### 2. 安装依赖
```bash
npm install
```

#### 3. 配置环境变量

复制配置文件：
```bash
# Windows
copy .env.example .env

# Linux/Mac
cp .env.example .env
```

编辑 `backend/.env` 文件：
```env
# 数据库配置（重要！）
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=campus_activities

# JWT密钥（建议修改）
JWT_SECRET=your-random-secret-key-here

# 其他配置保持默认即可
PORT=3000
NODE_ENV=development
CORS_ORIGIN=http://localhost:5173
```

#### 4. 初始化数据库
```bash
npm run init-db
```

成功后会显示：
```
✅ 数据库表创建成功
✅ 管理员账号创建成功 (admin/123456)
✅ 示例学生账号创建成功 (student1/123456)
✅ 示例活动创建成功
🎉 数据库初始化完成！
```

#### 5. 启动后端服务
```bash
npm run dev
```

看到以下信息表示成功：
```
🚀 校园活动管理系统后端API启动成功！
📡 服务器地址: http://localhost:3000
🏥 健康检查: http://localhost:3000/health
```

**保持这个终端运行！**

### 第三步：配置前端

#### 1. 新开一个终端，回到项目根目录

#### 2. 安装依赖

看到以下信息表示成功：

## 🐛 常见问题排查

### 问题1：后端启动失败

**错误信息**：`数据库连接失败`

**解决方法**：
1. 检查MySQL服务是否启动
   ```bash
   # Windows
   net start MySQL80
   
   # Mac
   brew services start mysql
   
   # Linux
   sudo systemctl start mysql
   ```

2. 检查 `backend/.env` 中的数据库配置
3. 确认数据库已创建

**错误信息**：`端口3000已被占用`

**解决方法**：
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Linux/Mac
lsof -i :3000
kill -9 <PID>
```

或修改 `backend/.env` 中的 `PORT=3001`

### 问题2：前端无法连接后端

**错误信息**：`Network Error` 或 `CORS错误`

**解决方法**：
1. 确认后端已启动（访问 http://localhost:3000/health）
2. 检查 `backend/.env` 中的 `CORS_ORIGIN=http://localhost:5173`
3. 清除浏览器缓存，刷新页面

### 问题3：登录后提示未授权

**解决方法**：
1. 打开浏览器开发者工具（F12）
2. 查看 Application → Local Storage
3. 确认有 `token` 和 `userInfo`
4. 如果没有，重新登录

### 问题4：数据不同步

**解决方法**：
1. 前端使用的是后端API，不再使用localStorage
2. 刷新页面查看最新数据
3. 检查浏览器控制台是否有错误

## 📊 数据流程

```
用户操作（点击报名）
    ↓
前端组件触发事件
    ↓
调用 Pinia Store 方法
    ↓
Store 调用 API (src/api/index.js)
    ↓
Axios 发送 HTTP 请求 (src/utils/request.js)
    ↓
后端接收请求 (src/routes/*.js)
    ↓
路由转发到控制器 (src/controllers/*.js)
    ↓
控制器处理业务逻辑
    ↓
操作数据库 (src/models/*.js)
    ↓
返回JSON响应
    ↓
前端接收响应
    ↓
更新 Store 状态
    ↓
组件自动更新UI
```

## 🎓 学习建议

### 1. 理解前后端分离

- **前端**：负责UI展示和用户交互
- **后端**：负责业务逻辑和数据存储
- **通信**：通过HTTP API（RESTful）

### 2. 查看代码结构

**后端核心文件**：
- `backend/src/app.js` - 应用入口
- `backend/src/routes/` - 路由定义
- `backend/src/controllers/` - 业务逻辑
- `backend/src/models/` - 数据模型

**前端核心文件**：
- `src/api/index.js` - API接口定义
- `src/utils/request.js` - HTTP请求配置
- `src/stores/` - 状态管理

### 3. 调试技巧

**后端调试**：
- 查看终端日志
- 使用 `console.log()` 打印变量
- 使用Postman测试API

**前端调试**：
- F12打开开发者工具
- Console查看日志
- Network查看API请求
- Vue DevTools查看组件状态
