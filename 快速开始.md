# 快速开始指南

## 🚀 一键启动（推荐）

### Windows 用户
双击运行 `start.bat` 文件，脚本会自动：
1. ✅ 检查 Node.js 环境
2. ✅ 安装项目依赖
3. ✅ 创建环境配置
4. ✅ 启动开发服务器

### Mac/Linux 用户
```bash
chmod +x start.sh
./start.sh
```

## 📝 手动启动

如果一键启动脚本无法运行，请按以下步骤手动启动：

### 1. 安装依赖
```bash
npm install
```

如果安装速度慢，使用国内镜像：
```bash
npm install --registry=https://registry.npmmirror.com
```

### 2. 启动项目
```bash
npm run dev
```

### 3. 访问系统
打开浏览器访问：`http://localhost:3000`

## 🤖 AI 功能配置（可选）

如果需要使用 AI 智能功能：

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/) 注册账号
2. 获取 API Key
3. 在项目根目录创建 `.env` 文件（或复制 `.env.example`）
4. 填入你的 API Key：
```env
VITE_DEEPSEEK_API_KEY=your_api_key_here
VITE_DEEPSEEK_API_URL=https://api.deepseek.com/v1
```

> 💡 提示：不配置 API Key 也可以正常使用系统，AI 功能会使用降级方案。

## ❓ 常见问题

### Q1: 提示 "npm 不是内部或外部命令"
**A:** 需要先安装 Node.js
- 下载地址：https://nodejs.org/
- 选择 LTS 版本（推荐 16.x 或更高）

### Q2: 依赖安装失败
**A:** 尝试以下方法：
```bash
# 方法1: 使用国内镜像
npm install --registry=https://registry.npmmirror.com

# 方法2: 清除缓存后重试
npm cache clean --force
npm install

# 方法3: 删除 node_modules 后重新安装
rm -rf node_modules package-lock.json
npm install
```

### Q3: 端口 3000 被占用
**A:** 修改 `vite.config.js` 中的端口号：
```javascript
server: {
  port: 3001,  // 改为其他端口
  open: true
}
```

### Q4: 页面空白或报错
**A:** 检查以下几点：
1. 确保所有依赖都已安装
2. 检查浏览器控制台的错误信息
3. 确认 Node.js 版本 >= 16
4. 尝试清除浏览器缓存

### Q5: AI 功能不工作
**A:** 
1. 检查 `.env` 文件中的 API Key 是否正确
2. 确认网络可以访问 DeepSeek API
3. 查看浏览器控制台是否有 API 调用错误
4. 不配置 API Key 时，AI 功能会使用本地降级方案
