# 代码保存指南

本指南将帮助您将项目代码保存到本地和GitHub。

---

## 📦 方法一：保存到本地（压缩备份）

### 步骤1：创建压缩包

**使用Windows资源管理器**：

1. 打开文件资源管理器
2. 导航到：`C:\Users\26036\.cursor\projects`
3. 右键点击 `d-campus-activities` 文件夹
4. 选择 **"发送到" → "压缩(zipped)文件夹"**
5. 重命名为：`d-campus-activities-backup-2026-01-31.zip`
6. 将压缩包移动到您想要的位置（如桌面、U盘等）

**或使用PowerShell命令**：

```powershell
# 打开PowerShell，运行以下命令

# 创建备份文件夹
$backupPath = "C:\Users\26036\Desktop\campus-activities-backup"
New-Item -ItemType Directory -Path $backupPath -Force

# 压缩项目
Compress-Archive -Path "C:\Users\26036\.cursor\projects\d-campus-activities\*" -DestinationPath "$backupPath\campus-activities-$(Get-Date -Format 'yyyy-MM-dd').zip" -Force

Write-Host "✅ 备份完成！文件保存在：$backupPath"
```

## 🌐 方法二：保存到GitHub

### 前置准备

1. **安装Git**（如果还没安装）
   - 下载：https://git-scm.com/download/win
   - 安装时选择默认选项即可

2. **注册GitHub账号**（如果还没有）
   - 访问：https://github.com
   - 点击 "Sign up" 注册

3. **配置Git**（首次使用）
   ```powershell
   git config --global user.name "你的名字"
   git config --global user.email "你的邮箱@example.com"
   ```

---

### 步骤1：创建GitHub仓库

1. 登录 https://github.com
2. 点击右上角的 **"+"** → **"New repository"**
3. 填写信息：
   - **Repository name**: `campus-activities-system`
   - **Description**: `智能校园活动管理系统`
   - **Public/Private**: 选择 Private（私有）或 Public（公开）
   - **不要勾选** "Initialize this repository with a README"
4. 点击 **"Create repository"**
5. 记下仓库地址（类似：`https://github.com/你的用户名/campus-activities-system.git`）

---

### 步骤2：初始化Git仓库并推送代码

**打开PowerShell或Git Bash**，运行以下命令：

```powershell
# 1. 进入项目目录
cd C:\Users\26036\.cursor\projects\d-campus-activities

# 2. 初始化Git仓库
git init

# 3. 添加所有文件到暂存区
git add .

# 4. 创建第一次提交
git commit -m "初始提交：智能校园活动管理系统完整代码"

# 5. 添加远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/campus-activities-system.git

# 6. 推送代码到GitHub
git push -u origin master
```

**注意**：
- 如果推送时要求输入用户名和密码，输入你的GitHub用户名和密码
- 如果GitHub要求使用Personal Access Token（个人访问令牌），请按照下面的步骤创建

---

### 步骤3：创建GitHub Personal Access Token（如果需要）

如果推送时提示需要token：

1. 访问：https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. 填写信息：
   - **Note**: `Campus Activities System`
   - **Expiration**: 选择过期时间（建议90天或更长）
   - **Select scopes**: 勾选 `repo`（完整的仓库访问权限）
4. 点击 **"Generate token"**
5. **复制生成的token**（只显示一次，请妥善保存）
6. 在推送时，用户名输入GitHub用户名，密码输入这个token

---

### 步骤4：验证上传成功

1. 访问你的GitHub仓库页面
2. 刷新页面
3. ✅ 应该能看到所有项目文件

## 📋 创建.gitignore文件（推荐）

为了避免上传不必要的文件，建议创建 `.gitignore` 文件：

**在项目根目录创建 `.gitignore` 文件，内容如下**：

```
# 依赖
node_modules/
backend/node_modules/

# 环境变量
.env
.env.local
.env.*.local

# 日志
npm-debug.log*
yarn-debug.log*
yarn-error.log*
pnpm-debug.log*
lerna-debug.log*

# 编辑器
.vscode/*
!.vscode/extensions.json
.idea
.DS_Store
*.suo
*.ntvs*
*.njsproj
*.sln
*.sw?

# 构建输出
dist/
dist-ssr/
*.local

# 数据库
backend/database.sqlite
*.sqlite
*.db

# 临时文件
*.log
.cache/
.temp/
```

**创建后运行**：

```powershell
git add .gitignore
git commit -m "添加.gitignore文件"
git push
```

## ❓ 常见问题

### Q1: 推送时提示 "fatal: remote origin already exists"

**解决方法**：
```powershell
git remote remove origin
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin master
```

### Q2: 推送时提示 "error: failed to push some refs"

**解决方法**：
```powershell
git pull origin master --allow-unrelated-histories
git push -u origin master
```

### Q3: 如何在新电脑上获取代码？

**方法**：
```powershell
# 克隆仓库
git clone https://github.com/你的用户名/campus-activities-system.git

# 进入目录
cd campus-activities-system

# 安装依赖
npm install
cd backend
npm install
cd ..

# 初始化数据库
cd backend
npm run init-db
cd ..

# 启动项目
# 终端1：启动后端
cd backend
npm run dev

# 终端2：启动前端
npm run dev
```

### Q4: 如何删除GitHub上的敏感信息？

如果不小心上传了密码等敏感信息：

1. 立即修改密码
2. 删除包含敏感信息的文件
3. 提交并推送
4. 如果需要完全删除历史记录，参考：https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/removing-sensitive-data-from-a-repository
