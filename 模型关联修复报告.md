# 模型关联修复报告

## ❌ 问题描述

**错误信息**：`Activity is not associated to Registration!`

**出现时机**：登录后加载用户数据时

**原因**：Registration 模型没有定义与 Activity 和 User 的直接关联关系

---

## 🔧 修复内容

**文件**：`backend/src/models/index.js`

### 修改前：
```javascript
// 只定义了多对多关系
User.belongsToMany(Activity, { through: Registration, ... })
Activity.belongsToMany(User, { through: Registration, ... })
```

### 修改后：
```javascript
// 先定义直接关联（一对多）
Registration.belongsTo(User, { foreignKey: 'userId', as: 'user' })
User.hasMany(Registration, { foreignKey: 'userId', as: 'registrations' })

Registration.belongsTo(Activity, { foreignKey: 'activityId', as: 'activity' })
Activity.hasMany(Registration, { foreignKey: 'activityId', as: 'registrations' })

Favorite.belongsTo(User, { foreignKey: 'userId', as: 'user' })
User.hasMany(Favorite, { foreignKey: 'userId', as: 'favorites' })

Favorite.belongsTo(Activity, { foreignKey: 'activityId', as: 'activity' })
Activity.hasMany(Favorite, { foreignKey: 'activityId', as: 'favorites' })

// 再定义多对多关系
User.belongsToMany(Activity, { through: Registration, ... })
Activity.belongsToMany(User, { through: Registration, ... })
```

## 📊 关联关系说明

### Registration 模型关联
```javascript
// Registration 属于 User（多对一）
Registration.belongsTo(User, { foreignKey: 'userId', as: 'user' })

// Registration 属于 Activity（多对一）
Registration.belongsTo(Activity, { foreignKey: 'activityId', as: 'activity' })
```

**用途**：
- 查询报名记录时可以包含用户信息
- 查询报名记录时可以包含活动信息

### Favorite 模型关联
```javascript
// Favorite 属于 User（多对一）
Favorite.belongsTo(User, { foreignKey: 'userId', as: 'user' })

// Favorite 属于 Activity（多对一）
Favorite.belongsTo(Activity, { foreignKey: 'activityId', as: 'activity' })
```

**用途**：
- 查询收藏记录时可以包含用户信息
- 查询收藏记录时可以包含活动信息

## 📝 技术说明

### Sequelize 关联类型

#### 1. belongsTo（属于）
```javascript
Registration.belongsTo(Activity)
// Registration 表中有 activityId 外键
// 可以通过 registration.getActivity() 获取关联的活动
```

#### 2. hasMany（拥有多个）
```javascript
Activity.hasMany(Registration)
// Activity 可以有多个 Registration
// 可以通过 activity.getRegistrations() 获取所有报名记录
```

#### 3. belongsToMany（多对多）
```javascript
User.belongsToMany(Activity, { through: Registration })
// 通过 Registration 中间表建立多对多关系
// 可以通过 user.getRegisteredActivities() 获取用户报名的所有活动
```

### 关联顺序很重要

必须先定义直接关联（belongsTo/hasMany），再定义多对多关联（belongsToMany），否则 Sequelize 无法正确解析关联关系。
