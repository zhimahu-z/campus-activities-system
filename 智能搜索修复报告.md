# 智能搜索修复报告

## 问题描述

在智能搜索页面，无论输入什么关键词，搜索结果都只显示两个固定的活动：
- 人工智能前沿讲座（ID: 1）
- 校园音乐节（ID: 2）

## 问题原因

### 根本原因
在 `src/services/deepseek.js` 文件中，`localIntelligentMatch()` 方法存在硬编码的活动ID：

```javascript
// ❌ 错误的实现
if (prompt.includes('搜索') || prompt.includes('search')) {
  const keywords = this.extractKeywords(prompt)
  return JSON.stringify({
    relevantIds: ['1', '2'],  // 硬编码的ID
    relatedKeywords: keywords,
    suggestedFilters: {
      category: keywords.length > 0 ? this.guessCategory(keywords[0]) : ''
    }
  })
}
```

### 问题链路

1. 用户在搜索页面输入关键词
2. 调用 `aiStore.enhanceSearch(query, activities)`
3. 调用 `deepseekService.enhanceSearch(query, activities)`
4. 内部调用 `this.chat(prompt, systemPrompt, 0.3)`
5. 当 DeepSeek API Key 未配置或API调用失败时，调用 `localIntelligentMatch(prompt)`
6. **返回硬编码的 `['1', '2']`**，导致只显示这两个活动

### 为什么会有这个问题？

- `enhanceSearch()` 方法本身有正确的降级逻辑（基于关键词匹配）
- 但 `chat()` 方法在API失败时会先调用 `localIntelligentMatch()`
- `localIntelligentMatch()` 返回了硬编码的ID，导致正确的降级逻辑没有执行

## 修复方案

### 修改1：优化 `enhanceSearch()` 方法

在方法开始时就检查API Key，如果未配置直接使用本地算法：

```javascript
async enhanceSearch(query, activities) {
  // 检查API Key是否配置
  const hasValidApiKey = DEEPSEEK_API_KEY && 
                        DEEPSEEK_API_KEY !== 'sk-your-api-key-here' && 
                        DEEPSEEK_API_KEY !== ''
  
  if (!hasValidApiKey) {
    console.warn('⚠️ DeepSeek API Key未配置，使用本地搜索算法')
    return this.localSearchAlgorithm(query, activities)
  }
  
  // ... API调用逻辑
  
  // API失败时降级
  return this.localSearchAlgorithm(query, activities)
}
```

### 修改2：提取本地搜索算法

将降级逻辑提取为独立方法 `localSearchAlgorithm()`：

```javascript
localSearchAlgorithm(query, activities) {
  console.log('🔄 使用本地搜索算法')
  const lowerQuery = query.toLowerCase()
  
  // 多维度匹配：标题、描述、标签、分类
  const matchedActivities = activities.filter(a => 
    a.title.toLowerCase().includes(lowerQuery) ||
    a.description.toLowerCase().includes(lowerQuery) ||
    a.tags.some(tag => tag.toLowerCase().includes(lowerQuery)) ||
    a.category.toLowerCase().includes(lowerQuery)
  )
  
  const keywords = this.extractKeywords(query)
  
  return {
    relevantIds: matchedActivities.map(a => a.id.toString()),
    relatedKeywords: keywords.length > 0 ? keywords : [query, '相关活动', '推荐'],
    suggestedFilters: {
      category: keywords.length > 0 ? this.guessCategory(keywords[0]) : '',
      tags: keywords
    }
  }
}
```

### 修改3：修复 `chat()` 方法

让 `chat()` 方法在搜索/推荐场景下抛出错误，而不是返回硬编码数据：

```javascript
async chat(prompt, systemPrompt = '你是一个智能助手', temperature = 0.7) {
  // 检查API Key是否配置
  if (!DEEPSEEK_API_KEY || DEEPSEEK_API_KEY === 'sk-your-api-key-here' || DEEPSEEK_API_KEY === '') {
    console.warn('⚠️ DeepSeek API Key未配置，使用本地智能匹配')
    // 对于需要具体数据的场景，抛出错误让调用方处理
    if (prompt.includes('搜索') || prompt.includes('search') || 
        prompt.includes('推荐') || prompt.includes('recommend')) {
      throw new Error('API_KEY_NOT_CONFIGURED')
    }
    return this.localIntelligentMatch(prompt)
  }
  
  try {
    // ... API调用
  } catch (error) {
    // 对于需要具体数据的场景，抛出错误让调用方处理
    if (prompt.includes('搜索') || prompt.includes('search') || 
        prompt.includes('推荐') || prompt.includes('recommend')) {
      throw error
    }
    return this.localIntelligentMatch(prompt)
  }
}
```

### 修改4：清理 `localIntelligentMatch()` 方法

移除硬编码的搜索和推荐逻辑：

```javascript
localIntelligentMatch(prompt) {
  console.log('💡 使用本地智能匹配算法')
  
  // 对于搜索、推荐等需要具体活动数据的场景
  // 应该抛出错误，让调用方使用自己的降级逻辑
  if (prompt.includes('搜索') || prompt.includes('search') || 
      prompt.includes('推荐') || prompt.includes('recommend')) {
    throw new Error('API不可用，请使用本地算法')
  }
  
  // 只处理评论分析、摘要提取、数据分析等不需要具体活动数据的场景
  // ...
}
```
