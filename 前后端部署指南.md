# 前后端分离部署指南

## 🎯 概述

本项目采用前后端分离架构：
- **前端**: Vue 3 + Vite（端口：5173）
- **后端**: Node.js + Express + MySQL（端口：3000）

## 📋 部署前准备

### 1. 环境要求

- Node.js >= 16.0.0
- MySQL >= 5.7
- npm 或 yarn

### 2. 检查环境

```bash
node -v
npm -v
mysql --version
```

## 🚀 后端部署步骤

### 步骤1：安装依赖

```bash
cd backend
npm install
```

### 步骤2：配置环境变量

复制 `.env.example` 为 `.env`：

```bash
cp .env.example .env
```

编辑 `.env` 文件，配置数据库信息：

```env
# 服务器配置
PORT=3000
NODE_ENV=production

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=campus_activities

# JWT密钥（请修改为随机字符串）
JWT_SECRET=your-random-secret-key-here
JWT_EXPIRES_IN=7d

# CORS配置
CORS_ORIGIN=http://localhost:5173
```

### 步骤3：创建数据库

登录MySQL：

```bash
mysql -u root -p
```

创建数据库：

```sql
CREATE DATABASE campus_activities CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

### 步骤4：初始化数据库

```bash
npm run init-db
```

这将创建所有表并插入初始数据：
- 管理员账号：admin / 123456
- 示例学生账号：student1 / 123456
- 3个示例活动

### 步骤5：启动后端服务

开发模式（自动重启）：
```bash
npm run dev
```

生产模式：
```bash
npm start
```

后端服务将在 `http://localhost:3000` 启动。

### 步骤6：测试后端API

访问健康检查接口：
```bash
curl http://localhost:3000/health
```

应该返回：
```json
{
  "status": "ok",
  "message": "校园活动管理系统API运行正常",
  "timestamp": "2026-01-30T12:00:00.000Z"
}
```

## 🔧 前端对接后端

### 修改说明

前端已经配置好了API对接，主要文件：

1. **`src/utils/request.js`** - Axios配置，自动添加token
2. **`src/api/index.js`** - 所有API接口定义
3. **`.env`** - 后端API地址配置

### 使用示例

在Store中使用API：

```javascript
import { userAPI, activityAPI } from '@/api'

// 登录
const login = async (credentials) => {
  try {
    const response = await userAPI.login(credentials)
    token.value = response.token
    userInfo.value = response.user
    localStorage.setItem('token', response.token)
    return { success: true }
  } catch (error) {
    return { success: false, message: error.message }
  }
}

// 获取活动列表
const fetchActivities = async () => {
  try {
    const response = await activityAPI.getAll({ page: 1, limit: 10 })
    activities.value = response.data
  } catch (error) {
    console.error('获取活动列表失败:', error)
  }
}
```

## 🐛 常见问题

### 1. 跨域问题

**问题**：前端请求后端时出现CORS错误

**解决**：
- 检查后端 `.env` 中的 `CORS_ORIGIN` 是否正确
- 确保后端已启动
- 清除浏览器缓存

### 2. 数据库连接失败

**问题**：后端启动时提示数据库连接失败

**解决**：
- 检查MySQL服务是否启动
- 检查 `.env` 中的数据库配置
- 确认数据库已创建

### 3. Token认证失败

**问题**：登录后访问其他接口提示未授权

**解决**：
- 检查localStorage中是否有token
- 检查token是否过期
- 重新登录获取新token

### 4. 端口被占用

**问题**：启动时提示端口已被占用

**解决**：
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# Linux/Mac
lsof -i :3000
kill -9 <PID>
```

或修改 `.env` 中的端口号。

## 🌐 生产环境部署

### 后端部署

1. **使用PM2管理进程**

```bash
npm install -g pm2
cd backend
pm2 start src/app.js --name campus-api
pm2 save
pm2 startup
```

2. **配置Nginx反向代理**

```nginx
server {
    listen 80;
    server_name api.yourdomain.com;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 前端部署

1. **构建生产版本**

```bash
npm run build
```

2. **部署到Nginx**

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/campus-activities/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://localhost:3000;
    }
}
```

3. **上传dist目录到服务器**

```bash
scp -r dist/* user@server:/var/www/campus-activities/dist/
```
