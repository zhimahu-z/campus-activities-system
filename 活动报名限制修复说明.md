# 活动报名限制修复说明

## 问题描述
原系统中，活动状态为"进行中"和"已结束"的活动仍然可以进行报名，这不符合实际业务逻辑。

## 修复方案

### 业务规则
**只有状态为"未开始"的活动才允许报名**

- ✅ **未开始** (upcoming)：可以报名
- ❌ **进行中** (ongoing)：不可报名
- ❌ **已结束** (ended)：不可报名

## 用户体验改进

### 1. 视觉反馈
- **未开始**：按钮显示"立即报名"，可点击（蓝色）
- **进行中**：按钮显示"进行中"，禁用状态（灰色）
- **已结束**：按钮显示"已结束"，禁用状态（灰色）
- **已报名**：按钮显示"已报名"，可点击（蓝色）

### 2. 交互反馈
当用户尝试报名非"未开始"状态的活动时：
- **进行中**：提示"活动正在进行中，无法报名"
- **已结束**：提示"活动已结束，无法报名"

### 3. 状态标识
所有页面的活动卡片都显示清晰的状态标识：
- 🟣 **未开始**：紫色渐变徽章
- 🔴 **进行中**：粉红色渐变徽章 + 脉冲动画
- ⚪ **已结束**：灰色徽章

## 技术细节

### 活动状态判断逻辑
```javascript
const getActivityStatus = (activity) => {
  const now = new Date()
  const startTime = new Date(activity.startTime)
  const endTime = new Date(activity.endTime)
  
  if (now < startTime) return 'upcoming'      // 未开始
  if (now >= startTime && now <= endTime) return 'ongoing'  // 进行中
  return 'ended'                               // 已结束
}
```
