### 2. 实现活动报名人数实时更新

#### 核心机制：全局事件监听

**修改文件：src/stores/user.js**

```javascript
const registerActivity = (activityId) => {
  // 禁止管理员报名
  if (userInfo.value?.role === 'admin') {
    return { success: false, message: '管理员账号不能报名活动' }
  }
  
  if (!registrations.value.includes(activityId)) {
    registrations.value.push(activityId)
    if (userInfo.value) {
      const userId = userInfo.value.id
      localStorage.setItem(`registrations_${userId}`, JSON.stringify(registrations.value))
    }
    // ✅ 触发全局事件，通知所有页面更新
    window.dispatchEvent(new CustomEvent('activity-registration-changed'))
    return { success: true }
  }
  return { success: false, message: '已经报名过该活动' }
}

const unregisterActivity = (activityId) => {
  const index = registrations.value.indexOf(activityId)
  if (index > -1) {
    registrations.value.splice(index, 1)
    if (userInfo.value) {
      const userId = userInfo.value.id
      localStorage.setItem(`registrations_${userId}`, JSON.stringify(registrations.value))
    }
    // ✅ 触发全局事件，通知所有页面更新
    window.dispatchEvent(new CustomEvent('activity-registration-changed'))
    return { success: true }
  }
  return { success: false, message: '未报名该活动' }
}
```

## 📊 实时更新机制流程图

```
用户报名/取消报名
    ↓
userStore.registerActivity() / unregisterActivity()
    ↓
更新 localStorage
    ↓
触发全局事件: window.dispatchEvent('activity-registration-changed')
    ↓
所有监听该事件的页面接收通知
    ↓
各页面调用 updateActivityParticipants()
    ↓
重新计算报名统计: getAllRegistrationsStats()
    ↓
更新活动参与人数: updateParticipantsCount()
    ↓
页面实时显示最新数据
```

## 💡 设计思路

### 为什么使用全局事件？
1. **解耦合**：各组件独立监听，不需要直接引用
2. **实时性**：事件触发即时，所有监听者立即响应
3. **简单性**：不需要复杂的状态管理
4. **可扩展**：新页面只需添加监听即可

### 为什么在多个地方检查管理员？
1. **前端验证**：提供即时反馈，提升用户体验
2. **Store验证**：确保数据层的安全性
3. **双重保障**：即使前端被绕过，Store层仍然拦截
